2003/11/28
----------------

1.8.4에 추가된 내용들

- 받는 큐가 가득찼을 때, 지금까지는 해당하는 연결을 끊거나
  (공용큐를 사용하지 않을 때) 서버를 내렸다.(공용큐를 사용할 때)
  사실 이것이 공용큐를 사용하지 않은 주요 이유이기도 했다.

  받는 큐가 가득찼을 때, 일시적으로 IOCPRecv를 멈추게 하고
  받는 큐가 비었을 때, 멈춘 IOCPRecv가 있다면 다시 시작하도록 한다.

  이를 위해서 몇 개의 함수(gmPseudoTCP::mf__SetOOB, gmPseudoTCP::mf__GetOOB)의 이름이 사라졌고
  이들 함수가 사용하는 DWORD값으로 접근하는 m__dwOOB가 플래그들의 집합으로 사용되었다.
  플래그를 적절히 사용하기 위하여 몇 개의 함수가 추가되었다.

  응용프로그램에서 특정클라이언트와의 연결이 끊어졌는 지를 확인할 때에

	DWORD	dwX = gmhTCP.mf__GetOOB();
	if (dwX = (DWORD) -1) {
		// 연결의 보내는 쪽을 닫는다.
	}

  대신에

	if (gmhTCP.mf__bDisconnect()) {
		// 연결의 보내는 쪽을 닫느다.
	}

  그리고, 받는 큐에 더 이상 메시지가 없을 때는, 혹시 라이브러리에서
  IOCPRecv가 멈추었는 지를 확인하고 만약 그렇다면 다시 이를 활성화시키는 코드를
  다음과 같이 추가한다.

	// 더 이상 받는 큐에 메시지가 없을 때
	if (gmhTCP.mf__bPaused()) {
		// 받는 큐가 가득찼기 때문에 일시적으로 IOCPRecv가 멈추었다.
		if(gmhTCP.mf__IOCPRecvContinue() < 0) {
			// 이 연결은 문제가 있다.
		}
	}
